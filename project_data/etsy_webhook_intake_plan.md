# EverMagic — Etsy Webhook Intake Plan (Option C)

**Status:** Planning — ready to build  
**Goal:** Replace static Tally link in Etsy thank-you message with an automated, personalized form link sent via email immediately after purchase  
**Created:** 2026-04-08

---

## Problem with Current Approach

The current fulfillment plan (etsy_launch_plan.md §4) has Etsy auto-sending a static Tally link in the order confirmation message. Issues:
- Customer must manually select theme (they already bought a specific theme listing — redundant)
- Customer must type their email (typo risk = failed delivery)
- Static link → same intake token for everyone, or no token pre-fill
- Fully manual on your end if anything goes wrong

---

## Solution: Etsy `order.paid` Webhook → n8n → Pre-filled Tally Link

### New Customer Flow

```
Customer buys on Etsy
    ↓
Etsy fires order.paid webhook → n8n (immediate)
    ↓
n8n fetches full receipt from Etsy API
    (gets: buyer email, listing_id → theme)
    ↓
n8n generates fresh intake token → stores in intake_tokens table
    ↓
n8n builds personalized Tally URL:
    ?email=buyer@email.com&theme=SPACE_HERO&intake_token=EVRM-XXXXXXXX
    ↓
n8n sends branded EverMagic email to buyer with that link
    ↓
Customer opens form — email + theme already filled
    Customer only fills: child name, age, gender, hobby, hero trait, photos
    ↓
Tally → existing n8n intake pipeline (UNCHANGED)
```

---

## Fields Pre-filled Automatically

| Tally Field | Source | Notes |
|-------------|--------|-------|
| `email` | Etsy receipt buyer_email | Eliminates typo risk |
| `theme` | Listing ID → theme map | Each of 5 listings maps to 1 theme |
| `package` | Hardcoded BASIC | Only one package at launch |
| `intake_token` | Generated by n8n | Fresh EVRM-XXXXXXXX per order |

Customer still fills: child_name, age, gender, glasses, hair_color, skin_tone, hobby, hobby_detail, signature_look, hero_trait, parent_message, photo_main, photo_extra, consent.

## New n8n Workflow: "EverMagic Etsy Intake"

### Nodes (in order)

1. **Webhook** — receives POST from Etsy at a dedicated URL
   - Method: POST
   - Auth: verify Etsy signature headers (`webhook-id`, `webhook-timestamp`, `webhook-signature`)
   - Only process `event_type === "order.paid"`

2. **Supabase — fetch refresh token** — read `etsy_refresh_token` from `envs` table

3. **HTTP Request — refresh access token** — POST to Etsy token endpoint
   - `grant_type: refresh_token`
   - `refresh_token: {{from envs}}`
   - Returns new `access_token` (1h expiry) + new `refresh_token`

4. **Supabase — update refresh token** — write new refresh token back to `envs`
   - This resets the 90-day expiry clock on every order → never expires in practice

5. **HTTP Request — fetch receipt** — GET `{{resource_url}}` from webhook payload
   - Header: `Authorization: Bearer {{access_token}}`
   - Returns full receipt: buyer email, listing details, etc.

6. **Code node — map listing → theme**
   ```js
   const LISTING_THEME_MAP = {
     '1234567890': 'SPACE_HERO',
     '1234567891': 'ENCHANTED_PRINCESS',
     '1234567892': 'FANTASY_HERO',
     '1234567893': 'ANIMAL_GUARDIAN',
     '1234567894': 'HOME_HELPER',
   };
   // Fill in real listing IDs after Etsy shop is live
   const listingId = receipt.transactions[0].listing_id;
   const theme = LISTING_THEME_MAP[listingId] || 'SPACE_HERO';
   const buyerEmail = receipt.buyer_email;
   ```

7. **Code node — generate intake token**
   ```js
   const token = 'EVRM-' + Math.random().toString(36).slice(2,10).toUpperCase();
   // Returns token string
   ```

8. **Supabase — insert intake token** — insert into `intake_tokens` table
   - `token`: generated token
   - `order_source`: `'etsy'`
   - `etsy_receipt_id`: receipt ID from webhook
   - `used`: false

9. **Code node — build Tally URL**
   ```js
   const TALLY_FORM_ID = 'YOUR_FORM_ID'; // e.g. 'wMd5Kx'
   const base = `https://tally.so/r/${TALLY_FORM_ID}`;
   const params = new URLSearchParams({
     email: buyerEmail,
     theme: theme,
     package: 'BASIC',
     intake_token: token,
   });
   const tallyUrl = `${base}?${params.toString()}`;
   ```

10. **Send Email (SMTP)** — branded EverMagic email to buyer
    - Subject: `Your EverMagic personalization form is ready ✨`
    - Body: see email template below

### Email Template (draft)

Subject: Your EverMagic personalization form is ready ✨

## Etsy OAuth Setup (prerequisite — do this first)

### One-time Setup Steps

1. Go to https://www.etsy.com/developers → create a new app
2. Set OAuth redirect URI (can be localhost for initial token grab)
3. Request scopes: `transactions_r` (read receipt data)
4. Complete OAuth flow once manually using a tool like Postman or a small script
5. Capture: `access_token`, `refresh_token`
6. Store `refresh_token` in Supabase `envs` table as key `etsy_refresh_token`
7. Store `client_id` and `client_secret` in n8n credentials store

### Token Lifecycle

- Access token: expires in 1 hour (refreshed inline by workflow on every order)
- Refresh token: expires in 90 days — but workflow writes new refresh token back to `envs` on every run, so clock resets with each order. No manual re-auth needed as long as at least one order arrives every 90 days.

### Etsy Webhook Subscription

After app is created, subscribe to `order.paid` via the Etsy webhook portal:
- Event: `order.paid`
- Callback URL: your n8n webhook node URL (n8n Cloud URL, publicly accessible)
- Signature verification: implement in n8n using the three Etsy headers

## Etsy Listing Type: Made-to-Order Digital

EverMagic listings must be set up as **made-to-order** digital listings, NOT instant downloads.

**Key difference:**
- Instant download: file uploaded to the listing, auto-delivered on purchase
- Made-to-order: **no file on the listing**, seller uploads the finished file per-order when completing it

This means:
- No placeholder/Welcome PDF needed on the listing — the listing publishes without any file ✓
- The existing listing (already created without a file) is correctly set up ✓
- After n8n generates the PDFs, the order must be "completed" in Etsy Shop Manager with the files attached

### How Etsy order completion works (manual)
1. Shop Manager → Orders → New tab
2. Check the order → Complete order
3. Upload up to 5 files (max 20MB each) — storybook PDF + coloring PDF + certificate PDF
4. Click Complete order → Etsy emails buyer: "Your file is ready to download"

### The Etsy API cannot automate this
The Etsy API v3 has no endpoint for uploading files to individual orders. This is a confirmed limitation (Etsy open-api GitHub discussion #1301). The "complete order with file upload" step is **manual only** via Shop Manager.

### Delivery model

| Layer | What | Who delivers | When |
|-------|------|-------------|------|
| EverMagic email (primary) | Storybook + coloring + certificate (Supabase links) | n8n pipeline | Automatically, ~24–48h after form submitted |
| Etsy order completion (secondary) | Same PDFs uploaded to Etsy order | Manual (you) | After n8n generates; gives buyer official Etsy download |

**Both layers deliver the same files.** The EverMagic email is faster and automatic. The Etsy completion is required for:
- Etsy marking the order as fulfilled (affects Star Seller metrics)
- Buyer having a permanent download in their Etsy account
- Etsy review prompt being triggered

### Automation options (future)
If order volume grows and manual Etsy completion becomes a bottleneck:
- **OrderFiles app** (orderfiles.app) — third-party Etsy integration that automates made-to-order file delivery. Paid service but designed exactly for this use case.
- Build a browser automation script (Playwright/Puppeteer) to complete orders — fragile but possible.
- For now: manual is fine at launch volume (5–25 orders/month takes minutes).
